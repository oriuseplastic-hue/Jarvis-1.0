## Development Guidelines
- Follow the existing code style (run `npm run lint` or equivalent).
- Write clear commit messages: `feat: add new option`, `fix: resolve crash on empty input`.
- Add tests where appropriate and keep them passing.

## Submitting a Pull Request

2. Open a PR on the original repo.
3. Fill out the template, linking any related issues (`Closes #12`).
4. Wait for CI checks to pass and a maintainer to review.
